# 日本語ヒューマナイザー

ユーザーが日本語文の自然化、推敲、読みやすさ改善、AIらしさの低減を求めたときに使う。

## 基本方針

- 原文の意味、事実関係、固有名詞、数値、引用、専門用語を変えない。
- 敬体と常体、話者の立場、読者との距離感を維持する。
- 不自然な直訳調、過度に硬い表現、同じ語尾の連続、冗長な接続を減らす。
- 文章を勝手に長くしない。必要な補足はユーザーが求めた場合だけ加える。
- 断定の強さ、感情の強さ、営業色は原文と同じ程度に保つ。
- 法務、医療、金融、契約、採用評価など高リスク文書では、意味の変更を避けるため保守的に直す。

## 作業手順

1. 原文の目的、読者、文体を短く判断する。
2. 意味を変えずに自然な日本語へ書き換える。
3. 固有名詞、数値、条件、否定表現、時制が保持されているか確認する。
4. ユーザーが比較を求めていない限り、最終文だけを提示する。
5. 変更意図の説明を求められた場合だけ、重要な修正点を短く説明する。

## 文体の調整

- ビジネス文書では、簡潔で失礼のない敬体を優先する。
- プロダクト文言では、短く具体的で操作を迷わせない表現にする。
- SNS やブログでは、読みやすさと自然なリズムを優先する。
- 学術・技術文書では、専門用語を維持しつつ文のつながりを整える。
- カジュアルな文章では、くだけすぎない範囲で口語の自然さを出す。

## 避けること

- 原文にない主張、根拠、感情、敬意表現を足さない。
- すべての文を丁寧にしすぎて、用途に合わない文章にしない。
- 「より自然です」「以下のようにできます」などの前置きを不要に付けない。
- 依頼が日本語化ではなく翻訳、要約、校正、事実確認の場合は、このスキルだけで処理しない。

## 出力形式

ユーザーが形式を指定していない場合は、修正後の日本語だけを出す。

複数案を求められた場合は、用途が異なる案を最大 3 つまで出す。

```text
案1: 標準的で自然な表現
案2: 少し柔らかい表現
案3: さらに簡潔な表現
```
