# Quick Rules — 日本語ヒューマナイズ高速処方

この文書は、一回の実行で検出、書き換え、自己監査を済ませるための高速処方である。`taxonomy.md` の P1/P2 中心パターンを、現場で使える「パターン」と「処方」に圧縮する。文章の出自判定には使わない。AI検出器回避も保証しない。

原則: 定義は短く、処方は一つに絞る。判断に迷う場合は `revision-playbook.md` を読む。

## Do-NOT

次は検出対象にも機械的な書き換え対象にも含めない。

- 固有名詞、製品名、モデル名、機関名、作品名
- 数値、日付、単位、統計式、化学式、法令番号
- 直接引用、URL、コード、CLI、API名、規格名
- 法律、医療、金融、契約、仕様の義務・禁止・条件表現
- 業界標準の英語略語: LLM、GPU、MCP、API など

## 過剰修正ガード

| 指標 | 扱い |
| --- | --- |
| 局所修正を超える大幅改稿 | 意味保持を再監査し、不要な言い換えを戻す。 |
| 本文全体の構成や主張順が変わる修正 | 確定書き換えを止め、`proposal_only` または候補提示にする。 |
| 事実・数値・固有名詞の変更 | 直ちに戻す。 |
| 原文にない比喩・事例・根拠の追加 | 直ちに戻す。 |
| 明示化の過剰 | 原文にない背景、指示対象、接続関係、強調を足した場合は戻す。 |
| ジャンルの変化 | 直ちに戻す。報告書をエッセイ調にしない。 |

## A. 翻訳調

| ID | パターン | 優先度 | 処方 |
| --- | --- | --- | --- |
| JA-TL-01 | `彼`、`彼女`、`それ`、`これら` が段落内で過剰 | P2 | 省略、固有名詞、役割名、名詞句へ戻す。視点が曖昧になる場合だけ残す。 |
| JA-TL-02 | 抽象名詞・無生物主語 + `示す`、`可能にする`、`もたらす`、`思い出させる`、`持つ` | P3 | 自然なら残す。不自然なら「〜から分かる」「〜を見ると〜思い出す」「〜は〜がある」へ述語化する。 |
| JA-TL-03 | `によって`、`により` + 受動態 | P2 | 主体が明確で責任関係が変わらない場合だけ能動へ。主体を伏せる必要、被害・迷惑受身、慣用的な受動なら残す。 |
| JA-RB-19 | `〜について`、`〜に関して`、`〜における` の連発 | P1 | `〜を`、`〜で`、`〜では`、`〜の` へ軽くする。制度語・論文語として必要なら残す。 |
| JA-TL-04 | `検討を行う`、`実施を行う`、`改善を実行する` | P1 | 用語化していなければ `検討する`、`実施する`、`改善する` にする。 |
| JA-TL-05 | `することができる`、`することが可能` | P1 | 能力・許可・制度上の可能性でなければ `できる`、または現在形へ短くする。 |
| JA-TL-07 | 名詞前の長い連体修飾、関係代名詞風の左詰め修飾 | P2 | 文を分ける。後置説明、同格、短い補足にする。 |
| JA-MT-18 | 原文語順、代名詞、態、名詞句がそのまま残る | P2 | 原文対応を一度外し、日本語の焦点、省略、語順で再構成する。 |
| JA-MT-18 | 原文にない指示対象、背景説明、接続関係、強調を補っている | P2 | 明示化が必要か監査する。原文にない補足は本文へ混ぜず、必要なら注記候補に分ける。 |

## B. 英語・外来語・括弧

| ID | パターン | 優先度 | 処方 |
| --- | --- | --- | --- |
| JA-TL-06 | 未定着カタカナ語が多い | P3 | 読者に定着していなければ和語・漢語へ言い換える。専門語なら説明を添える。 |
| JA-TL-06 | 日本語 + 英語括弧を毎回併記 | P2 | 初出だけ併記し、以後は日本語または定着した略語だけにする。 |
| JA-TL-06 | 英語の名詞句をそのまま漢語で固める | P2 | 名詞連鎖をほどき、日本語の述語で「何がどうなるか」を示す。 |
| JA-TL-06 | `アジェンダ`、`ガイドライン`、`モニタリング`、`パブリックコメント` が読者に重い | P3 | 文脈に応じて `議題`、`指針`、`継続監視`、`意見募集` へ。制度名・固有名なら残す。 |
| JA-SP-23 | 専門語を自然化のために削っている | P2 | 専門語は残す。読者が非専門家なら、初出だけ短い説明を添える。 |

## C. 専門・学術翻訳

| ID | パターン | 優先度 | 処方 |
| --- | --- | --- | --- |
| JA-MT-22 | `可能性がある`、`示唆される`、`と考えられる` を断定・否定へ寄せる | P2 | 事実性、確信度、否定、条件、スコープを保つ。慎重表現の重複だけを畳む。 |
| JA-SP-23 | 同一概念の訳語、表記、略語が揺れる | P2 | 用語集、既存訳、分野慣習、初出略語を優先する。見た目のために言い換えない。 |
| JA-SP-23 | 用語の不統一、全半角、数字・日付・単位書式の揺れ | P2 | 意味を変えず、文書内の表記規則へ合わせる。ライトポストエディット相当なら対象範囲を確認する。 |
| JA-RB-19 | `しないわけではない`、`不可能ではない` などの二重否定・準否定 | P1 | 法務・研究の慎重表現でなければ素直にする。意味が弱まる場合は残す。 |
| JA-RB-19 | `〜することとする`、`〜ということになる`、同じ名詞の重複 | P1 | 義務・制度語でなければ `〜する`、`〜である`、重複をまとめる。 |

## D. 構造的なAI風

| ID | パターン | 優先度 | 処方 |
| --- | --- | --- | --- |
| JA-AI-11 | `まず`、`さらに`、`一方で`、`最後に` の型化 | P2 | 1〜2個だけ残し、他は文の流れへ溶かす。因果や対比を推測で足さない。 |
| JA-AI-14 | `このように`、`結論として`、`まとめると`、`今後も注目されることが期待されます` の締め | P2 | 原文内の判断、条件、限界だけで閉じる。総論だけの締めは削る。`期待されます` を `期待されています` に変えるだけで終えない。対象が明示されているなら `今後も関心を集めそうです`、`関心は今後も続きそうです` のように、定型句を外して述語を立て直す。 |
| JA-AI-13 | 箇条書き、太字、コロン見出しが過剰 | P2 | 実務手順なら残す。コラム、メール、散文なら段落へ戻す。 |
| JA-AI-12 | `重要`、`効果的`、`包括的`、`不可欠` が多い | P3 | 強調語を削り、原文にある対象・条件・理由を前に出す。根拠は足さない。 |
| JA-AI-15 | 具体性不足を一般論で埋めている | P3 | 原文にある人、時期、対象、手順だけを前面化する。新しい事実は追加しない。 |
| JA-LLM-20 | `AもBも重要`、`バランスが大切` で無難に逃げる | P3 | 文章内にある立場、制約、次の行動だけを残す。立場を捏造しない。 |

## E. リズム・係り受け

| ID | パターン | 優先度 | 処方 |
| --- | --- | --- | --- |
| JA-WR-09 | 文長が均一、または 100字超の長文が続く | P1 | 意味の切れ目で分ける。必要なら短文と長文を混ぜる。 |
| JA-WR-10 | 読点が機械的、または係り受けを助けていない | P2 | 読点数ではなく、主語・述語、修飾・被修飾の距離、節境界で打ち直す。 |
| JA-WR-08 | `の` の連続、同じ助詞の近接反復 | P1 | 動詞化、語順変更、名詞結合、文分割でほどく。 |
| JA-WR-21 | 敬体、常体、である調が不規則に混ざる | P1 | 引用、表、UIラベルを除き、用途に合う文体へ統一する。 |
| JA-WR-09 | 主語と述語が遠い、述語が最後まで見えない | P1 | 主述を近づける。複数の意味に取れる場合は語順を直し、長い修飾節は先に出すか分ける。 |
| JA-WR-09 | 一文に論点が複数あり、`が` や並列表現が続く | P1 | 一文一論点を目安に分ける。3項目以上の並列は文か箇条書きで整理する。 |
| JA-RB-19 | `理解することができない状態になった` のように語の結びつきが硬い | P1 | `理解できなくなった`、`希望が持てなくなった` のように自然な連語へ畳む。意味を削らない。 |

## F. post-editese 監査

Toral 2019 の post-editese 研究は、機械翻訳後編集文に simplification、normalisation、interference が残りやすいことを示す。日本語では断定せず、監査信号として使う。

| ID | パターン | 優先度 | 処方 |
| --- | --- | --- | --- |
| JA-MT-16 | 短くしすぎて語感、皮肉、時代感が落ちる | P3 | 必要なニュアンスを戻す。短文化だけを目的にしない。 |
| JA-MT-17 | 無難で標準的な表現へ寄りすぎる | P3 | ジャンル固有の語り口、距離感、文の揺れを戻す。 |
| JA-MT-18 | 原文の sentence length、情報順、態に引きずられる | P2 | 日本語の自然な焦点化、省略、語順へ組み直す。 |
| JA-LLM-20 | 翻訳後の文が滑らかでも声が平板 | P3 | 話者、読者、媒体、文脈に合う硬さへ戻す。装飾は足さない。 |

## G. 小説・随筆・現代文

| パターン | 優先度 | 処方 |
| --- | --- | --- |
| 抽象名詞句を直訳して硬い | P2 | `fine health` を「上等な健康」で止めず、「元気」「健やかさ」「体に満ちる感じ」など文脈に合わせる。 |
| 比喩の像が名詞で固まる | P2 | `bursts of leaves` は「葉のかたまり」ではなく、「葉が一斉に吹き出す」「葉が勢いよく広がる」など動きで訳す。 |
| 時間表現を狭く取りすぎる | P2 | `one year` を文脈なしに「一年生のとき」と決めず、「ある年」「一年のあいだ」に留める。 |
| 皮肉を含む人物評が説明語になる | P2 | `well-rounded man` は「バランスの取れた人間」だけで済ませず、語りの皮肉に応じて「幅広い教養人」などを検討する。 |
| 人物語が粗い | P2 | `man`、`woman` は文脈に応じて「男性」「女性」「人物」「男」「女」を選ぶ。原文より粗くしない。 |
| 役割語・女ことばを補いすぎる | P2 | 原文にない `わ`、`わよ`、`かしら`、`ぜ`、`やあ`、`〜さ`、`〜かい`、`〜だい` などで性別、年齢、階層、性格を足さない。話者像の根拠がある場合だけ使う。 |

## H. 自己検証

書き換え直後に確認する。

1. 固有名詞、数値、日付、単位、引用、URL、コードを変えていないか。
2. 義務、禁止、条件、否定、責任主体を変えていないか。
3. 局所修正を超える大幅改稿になっていないか。本文全体の構成や主張順が変わるなら確定書き換えを止める。
4. 原文にない比喩、体験、根拠、専門知識を足していないか。
5. 明示化で原文にない背景、指示対象、接続関係、強調を足していないか。
6. 役割語や文末詞で人物像を勝手に変えていないか。
7. 事実性、確信度、否定、条件、スコープを変えていないか。
8. 用語の不統一、表記ゆれ、初出略語の不統一を残していないか。
9. P1 の安全な表層修正が残っていないか。
10. P2/P3 を機械的に直して意味を狭めていないか。
11. 文体、読者、媒体、ジャンルが変わっていないか。

## 根拠の使い方

詳細な出典の位置付けは `evidence.md` を読む。ここでは Zaitsu、Toral、AAMT、JTF、JIS、文化庁、国立国語研究所、textlint、UTX、function word、明示化、役割語、事実性、用語の不統一、ジャンル差、読点位置、連語などの研究・実装参考を、出自判定ではなく処方の優先順位を決める補助信号としてだけ使う。
