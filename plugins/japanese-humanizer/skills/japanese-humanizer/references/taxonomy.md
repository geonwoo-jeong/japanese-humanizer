# 日本語文体改善 taxonomy

この taxonomy は、AIが作成した日本語文章や翻訳調の文章に残りやすい、AIっぽい均質さ、MTese、post-editese、冗長で平板な文体を、説明可能に改善するための分類である。文章の出自判定には使わない。AI検出器の通過も保証しない。

## 根拠 Grade

| Grade | 意味 |
| --- | --- |
| A | 日本語を直接対象にした査読論文、学術論文、公的研究、公的資料 |
| B | 日本語を直接対象にした大学紀要、学会発表、専門家資料、学位論文、公的校正資料 |
| C | 多言語研究、英語・欧州言語中心だが日本語へ慎重に拡張できる研究 |
| D | textlint、GitHub、公式ツール仕様などの実装参考 |
| E | 実務上の仮説。日本語での直接検証は未確認 |

## 使用原則

- ID は修正理由の説明に使う。機械的な置換命令ではない。
- AI生成文として渡された文章では、AIっぽさの低減を主目的にしつつ、意味保持を最優先する。
- `speculative: true` の項目は、ユーザーに断定的に説明しない。
- 誤検出リスクが `high` の項目は、本文へ反映する前に文脈を確認する。
- 法務、医療、研究、仕様、規約では、自然さより意味保持を優先する。

## 実行優先度

この taxonomy は検出器ではなく、修正順を決める実行ルーブリックとして使う。

| 優先度 | 意味 | 重み | 操作 |
| --- | --- | --- | --- |
| P1 | 研究・実装根拠が強く、意味保持しやすい表層修正 | 4 | まず直す。ただし高リスク文書では候補に留める。 |
| P2 | AI風・翻訳調の印象に効くが、文脈判断が必要 | 3 | 文章全体の流れを見て直す。原文にない情報は足さない。 |
| P3 | 誤検出が多い、または高次判断が必要 | 1 | スコアリングと監査に使い、本文反映は慎重にする。 |

`修正優先度スコア` は、各指摘の重みを合計して 0〜100 に丸めた作業量の目安である。文章の出自判定、AI検出器回避、品質保証には使わない。

## ルールベースで強く検出する項目

| ID | 検出対象 | 代表例 | 注意 |
| --- | --- | --- | --- |
| RB-01 | 文長 | 100字超、読点4個以上 | ジャンルにより許容値を変える。 |
| RB-02 | 「することができる」 | 「利用することができます」 | 可能・許可・能力の意味差を見る。 |
| RB-03 | サ変名詞＋行う/実施/実行 | 「調査を実施する」 | 行政・契約では必要な場合がある。 |
| RB-04 | 受動態＋「によって」 | 「委員会によって決定された」 | 主体をぼかす必要がある場合は維持する。 |
| RB-05 | 「の」連続 | 「制度の変更の影響」 | 固有名詞・専門語の一部は除外する。 |
| RB-06 | 同じ接続詞の近接反復 | 「しかし…しかし…」 | 修辞上の反復は例外。 |
| RB-07 | 外来語候補 | 「アジェンダ」「インキュベーション」 | 定着語は残す。 |
| RB-08 | 文末文体混在 | 「です・ます」と「である」 | 引用、表、箇条書きは除外する。 |
| RB-09 | 二重否定 | 「しないわけではない」 | 法的慎重表現は維持する場合がある。 |
| RB-10 | 括弧内長文 | 本文中の長い括弧説明 | 専門定義は保持する。 |
| RB-11 | 公用文・技術文の回りくどさ | 「〜することとする」「〜ということになる」 | 義務・制度語なら保持する。 |

## スコアリングだけ行う項目

- 機能語偏り: 「の」「に」「こと」「また」「そして」などの段落単位の偏りを見る。
- 接続表現パターン: 「まず / さらに / 一方で / このように」の型化を見る。
- 文長分散: 極端に均一、または極端に長い箇所を警告する。
- 句読点位置: 読点数ではなく、係り受けを助けているかを判断する。
- 抽象語密度: 「重要」「効果的」「可能性」などを段落単位で数える。
- 具体語密度: 人、場所、時期、対象、数値、手順があるかを確認する。
- 事実性・モダリティ: 断定、推量、否定、条件、スコープが変わっていないか確認する。
- 専門用語統一: 同一概念の訳語、表記、略語が文書内で揺れていないか確認する。
- 連語の自然さ: 直訳調の名詞句や「状態になった」型が、日本語の自然な述語にできるか確認する。
